## ADDED Requirements

### Requirement: Image input accepts common image formats
The system SHALL accept JPEG, PNG, and WebP image formats for family member recognition.

#### Scenario: Import JPEG photo
- **WHEN** user provides a .jpg/.jpeg image file
- **THEN** system loads and processes the image successfully

#### Scenario: Import PNG photo
- **WHEN** user provides a .png image file
- **THEN** system loads and processes the image successfully

### Requirement: Image recognition identifies visible people
The system SHALL use a vision LLM to analyze the image and output descriptions of visible family members including gender, age estimate, and apparent attributes.

#### Scenario: Single person photo
- **WHEN** user provides a photo containing one person
- **THEN** system outputs a description like "照片中有一位男性，看起来40多岁，穿着一件蓝色衬衫"

#### Scenario: Multiple people photo
- **WHEN** user provides a photo containing multiple people
- **THEN** system outputs descriptions for each person identified

### Requirement: Image is resized before vision API call
The system SHALL resize images to a maximum of 2048 pixels on the longest edge before sending to the vision model.

#### Scenario: Large image is resized
- **WHEN** user provides a 4000x3000 pixel image
- **THEN** system resizes it to 2048x1536 before API call

#### Scenario: Small image is unchanged
- **WHEN** user provides a 800x600 pixel image
- **THEN** system sends it without resizing

### Requirement: Vision output is processed through classify
The text description generated by the vision model SHALL be passed to the existing classify node for attribute extraction and storage.

#### Scenario: Vision output fed to classify
- **WHEN** vision model outputs "照片中有一位女性，看起来30多岁，长发，穿着一件白色连衣裙"
- **THEN** system passes this text to classify which extracts: member_name, attribute_type, content for ChromaDB storage

### Requirement: User confirms before adding recognized info
After vision model generates descriptions, the system SHALL display them to the user for confirmation before adding to the knowledge base.

#### Scenario: User approves recognition results
- **WHEN** system displays recognized descriptions and user confirms with "是" or "确认"
- **THEN** system adds the information to ChromaDB

#### Scenario: User rejects recognition results
- **WHEN** system displays recognized descriptions and user responds with "否" or "取消"
- **THEN** system discards the results and returns to input mode selection

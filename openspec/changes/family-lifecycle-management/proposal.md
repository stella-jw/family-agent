## Why

家庭信息散落在聊天记录、笔记本、记忆碎片中，难以系统化管理。随着时间推移，家庭成员的信息持续变化：
- 孩子的兴趣爱好随年龄改变
- 老人的健康指标需要持续监控
- 个人的工作、健康、心情每天都在更新

现有工具无法解决**长期追踪、多维对比、变化洞察**的需求。

## What Changes

**Family-Agent 从"信息记录工具"升级为"家庭生命周期管理系统"**

### 新增核心能力

| 能力 | 说明 |
|------|------|
| **时间序列追踪** | 每次更新带时间戳，支持历史查询 |
| **多维度分析** | 横向（时间轴）vs 纵向（维度）分析 |
| **里程碑事件** | 升学、结婚、生子、就医等重大节点 |
| **健康指标监控** | 血压、血糖、尿酸等指标的长期追踪 |
| **可视化报告** | 成长曲线、健康趋势、人生时间轴 |
| **AI 变化洞察** | 自动发现异常变化、预测风险 |

### 保留并增强现有功能

- 自然语言输入（文本/文件/图片/语音）
- 智能关系存储（双向关系网络）
- 代词消解（他/她上下文关联）

## Capabilities

### New Capabilities

- `lifecycle-tracking`：时间序列信息追踪，支持历史版本查询
- `health-monitoring`：健康指标监控与趋势分析
- `milestone-events`：人生里程碑事件记录（升学/结婚/生子/去世等）
- `visualization`：数据可视化报告（图表/曲线/时间轴）
- `change-insight`：AI 驱动的变化检测与洞察

### Modified Capabilities

- `family-agent`：增强为支持时间维度的生命周期管理

## Impact

- **时序数据模型**：需要扩展 ChromaDB schema 支持时间索引
- **可视化模块**：新增图表组件（Vue/ECharts）
- **分析引擎**：新增变化检测算法
- **报告生成**：新增定期报告（周/月/季/年）生成器

## 里程碑规划

| 阶段 | 目标 | 核心功能 |
|------|------|----------|
| Phase 1（当前） | 基础信息记录 | 多模态输入、双向关系、智能解析 |
| Phase 2 | 时序追踪 | 时间戳存储、历史查询、变化检测 |
| Phase 3 | 健康监控 | 健康指标追踪、趋势图表、异常预警 |
| Phase 4 | 人生里程碑 | 重大事件记录、人生时间轴、报告生成 |
| Phase 5 | AI 洞察 | 智能分析、趋势预测、个性化建议 |
